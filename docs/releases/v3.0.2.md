# v3.0.2

Yayin tarihi: 2026-04-23

## Ozet

Bu patch surum, loglarda gorulen iki gereksiz alarm sinifini hedef alir: backup klasoru baglaminda yanlis yerde aranan `proje_listesi.xlsx` uyarilari ve ayni eksik Excel dosyasi icin tekrar tekrar basilan warning satirlari. Ayni zamanda async proje yukleme watchdog log seviyesi gercek hata ile gecikme durumunu ayiracak sekilde netlestirildi.

## Degisiklikler

- `ExcelLoaderService`, istenen yol bulunamiyorsa ayni dosya adini ust dizinlerde arayarak en yakin mevcut `proje_listesi.xlsx` dosyasina fallback yapar.
- Excel dosyasi eksikse bos sonuc cache'lenir; ayni dosya icin her sorguda yeniden warning basilmaz.
- Proje yukleme watchdog'i, arka plan thread halen calisiyorsa warning yerine info log yazar; async yukleme gercekten bos veya basarisizsa warning korunur.
- Excel loader fallback ve watchdog log davranisi icin regression testleri eklendi.

## Analiz

Gozlenen warning'ler iki farkli kok nedenden geliyordu:

- Backup klasoru altindan acilan DB baglaminda `proje_listesi.xlsx` ayni klasorde araniyor, fakat dosya gercekte uygulama veya ana veritabani klasorunde duruyordu.
- Excel loader eksik dosyada erken donuyor ama bos sonucu cache'lemiyordu; bu nedenle her `find_project()` cagrisi ayni warning'i yeniden uretiyordu.

Bu surumle hem path cozumu daha dayanikli hale geldi hem de ayni eksik dosya icin log spam'i kesildi.
