# v2.1.9.1

Yayin tarihi: 2026-04-18

## Ozet

Bu surum, `v2.1.9` uzerine stabilite ve geriye uyumluluk sertlestirmesi getirir. Ozellikle cok kullanicili SQLite kullanimi, writer lease kurtarma davranisi ve eski yedek veritabanlarinin geri yuklenmesi sonrasi schema uyumlulugu guclendirildi.

## Degisiklikler

- Cok kullanicili writer lease akisi gecici heartbeat/ag hatalarinda hemen write dusurmeyecek sekilde esikli hale getirildi.
- Basarisiz login veya yarim auth akisi sonrasinda session kaydi ve writer lock temizligi garanti altina alindi.
- Writer lease kaybi algilandiginda uygulama kontrollu olarak salt okunur moda dusuruluyor ve arayuz write durumunu gorunur sekilde guncelliyor.
- Veritabani degistirme akisinda atomik geri donus guclendirildi; yeni DB tam hazir olmadan eski baglanti birakilmiyor.
- Legacy backup restore sonrasi `revizyonlar` OCR metadata kolonlari ile `yazi_dokumanlari` migrationi tekrar uygulanarak eski DB uyumlulugu guclendirildi.
- OCR/PDF okuma yolu, Tesseract backend ve belge zeka servisi regression testleriyle tekrar dogrulandi.
