# v3.0

Yayin tarihi: 2026-04-23

## Ozet

Bu surum, iki ayri dagitim riskini tek hatta kapatir: Windows'ta bazi bilgisayarlarda `.exe` dosyasinin yalnizca "Yonetici olarak calistir" ile acilabilmesine yol acan yazma yolu tasarimi duzeltildi ve legacy kaynak dosyalarda biriken Turkce karakter bozulmalari kaynaginda temizlendi.

## Degisiklikler

- Paketli surum artik log, guncelleme notlari, varsayilan veritabani ve yedekleri `%LOCALAPPDATA%\\BOTAS\\ProjeTakipSistemi` altinda tutuyor.
- Eski portable dagitimlarda `.exe` yanindaki `projeler.db`, ilk acilista kullanici profiline kopyalanarak veri kaybi olmadan yeni yapiya aliniyor.
- `main_window.py`, `database.py` ve kritik dialog akislari dahil legacy mojibake metinler kaynakta temiz UTF-8 metne cevrildi.
- Repo seviyesinde `.editorconfig`, `.gitattributes`, VS Code ayarlari ve UTF-8 terminal yardim script'i eklendi.
- Encoding hijyeni icin kaynak tarama testi eklendi; login dialogunda gercek Turkce karakter kullanimina test korumasi kondu.

## Analiz

Sorun iki katmanda gorunuyordu:

- Runtime tarafta uygulama bazi dosyalari `.exe` yanina yazmaya calisiyor, bu da korumali klasor veya kurumsal guvenlik politikasinda standart kullanici acilisini kirabiliyordu.
- Kaynak tarafta eski dosyalarin bir kismi tarihsel olarak mojibake ile repoya girmis, yeni ekranlarin bir kismi bunu runtime onarim katmaniyla yamaliyordu.

Bu surumde hem yazma yollarinin kullanici profiline tasinmasi hem de bozuk metinlerin dogrudan kaynakta temizlenmesi ile iki risk birlikte kapatildi.

## Dagitim Notu

Imzasiz `PyInstaller --onefile` dagitiminda antivirus veya antimalware false-positive riskini sifira indirmek teknik olarak garanti edilemez. Bu surum admin gereksinimini kaldirir, kaynak metinleri temizler ve UTF-8 gelistirme hijyenini kuvvetlendirir; daha ileri dagitim guveni icin code signing halen onerilir.
